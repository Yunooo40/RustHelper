# Phase 4.3 — K/D stats & leaderboard — Design

Date : 2026-06-22 · Sur `main` (push direct, pas de branche) · Dépend de Phase 4.2 (table `deaths`)

## Objectif
Exploiter la table `deaths` (déjà alimentée par `POST /webhook/death`) pour offrir des
**stats K/D par joueur** (`/stats`) et un **classement du serveur** (`/leaderboard`).
Aucune nouvelle ingestion : pur calcul de lecture. **Le plugin Rust ne change pas.**

## Choix produit — définitions K/D
Cohérentes entre `/stats` et le leaderboard :
- **Kill** = ligne `deaths` où `killer_id` = le joueur **et** `killer != victim`
  (un suicide / dégât auto ne donne jamais de kill).
- **Death** = toute ligne où `victim_id` = le joueur (PvP, NPC, environnement, suicide —
  c'est « combien de fois tu es mort »).
- **K/D** = `deaths > 0 ? kills / deaths : kills` — **jamais de division par zéro / ∞** ;
  un joueur invaincu a un ratio égal à son nombre de kills.
- **Best streak** = plus longue série de kills consécutifs sans mourir, sur la timeline
  chronologique (kills + morts) du joueur.

Autres choix :
- **Clé = Steam id** (stable) → tout l'historique compte, même les morts enregistrées
  avant que le joueur ne lie son compte Discord.
- **MVP = stats globales** (tous serveurs ; en pratique un seul serveur Rust par bot).
  Le filtre `serverId` existe déjà côté modèle pour le futur multi-serveur.
- `/stats` exige un compte **lié** (`/link`) car les stats sont indexées par Steam id.

## Modèle `backend/models/stats.js` (lecture seule)
- `kd(kills, deaths)` — ratio borné (pas d'∞).
- `forPlayer({ steamId, serverId? })` → `{ steamId, name, discordId, kills, deaths, kd, bestStreak }`.
  - `kills`/`deaths` via `COUNT`. `name`/`discordId` = la valeur la plus récente vue.
  - `bestStreak` calculé en JS sur la timeline ordonnée par `id`.
- `leaderboard({ serverId?, limit=10, minKills=0 })` → `[{ steamId, name, discordId, kills, deaths, kd }]`.
  - Agrégation par Steam id (dernier nom/discord gagne), tri `kd desc, kills desc`, `minKills` filtre.

## Commandes
- `/stats [joueur]` — joueur optionnel (défaut = l'invocateur). Résout Discord→Steam via
  `Link.findByDiscord`. Pas lié → message éphémère « /link ». Sinon `statsEmbed`
  (mention non-pingante via `allowedMentions:{parse:[]}`).
- `/leaderboard` — top 10 par K/D (`minKills:1`). `leaderboardEmbed` (médailles 🥇🥈🥉).

## Embeds (`bot/lib/embeds.js`)
- `statsEmbed({ name, discordId, kills, deaths, kd, bestStreak, serverName? })` — carte
  joueur (fields inline Kills / Deaths / K/D / Best streak).
- `leaderboardEmbed({ serverName?, rows })` — liste numérotée `🥇 **nom** — K/D · kills–deaths`.

## Tests (`tests/backend/stats.test.js`, 12)
Comptage kills/deaths · suicide exclu des kills mais compté en death · mort NPC (killer null) ·
K/D zéro-division · K/D standard · best streak (+ cassé par suicide) · filtre serverId ·
name/discord le plus récent · leaderboard tri+limit · minKills · suicides non comptés.
