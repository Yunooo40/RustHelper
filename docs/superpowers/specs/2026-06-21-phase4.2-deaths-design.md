# Phase 4.2 — Death notifications (kill feed) — Design

Date : 2026-06-21 · Branche : `feat/phase4.2-deaths` (depuis `main`, inclut 4.1)

## Objectif
Quand un joueur meurt en jeu, poster un **kill feed** dans le canal Discord du serveur,
et **@mentionner** les joueurs liés (Phase 4.1) — c'est le payoff concret du linking.

## Choix produit
- **Kill feed dans le canal** (pas de DM par mort : trop spammy en Rust). DM perso =
  option future.
- **Opt-in à la source** : le plugin a un toggle `Relay player deaths` (**défaut false**).
  L'API reste générique et traite ce qu'on lui envoie.
- Le **mention** Discord n'apparaît que pour un joueur lié ; sinon on affiche le pseudo.

## Flux
1. Plugin (`OnPlayerDeath`, opt-in) → POST `/webhook/death`
   `{ server, victim_id, victim_name, killer_id?, killer_name?, cause?, distance? }`.
2. API : valide, résout `victim_discord_id`/`killer_discord_id` via `Link.findBySteam`,
   persiste dans `deaths`, émet `DEATH_EVENT` sur le bus.
3. Bot : embed kill feed dans `channel_id` ; mention `<@discord_id>` si lié.

## Schéma (db.js) — additif
```sql
deaths (
  id PK, server_id FK->servers ON DELETE CASCADE,
  victim_id, victim_name, killer_id, killer_name, cause, distance REAL,
  victim_discord_id, killer_discord_id,   -- résolus à l'insert (dénormalisé, historique)
  payload, created_at
)
```

## Modèle `backend/models/death.js`
- `insert({ serverId, victimId, victimName, killerId, killerName, cause, distance, victimDiscordId, killerDiscordId, payload })`.
- `recent({ serverId, limit=25 })`.

## Route `POST /webhook/death` (auth webhook)
- `server` + `victim_name` requis → sinon **400**.
- Résout les liens (victime/tueur) via `Link.findBySteam(steam_id)`.
- `Servers.findOrCreateByName(server)`, insert death.
- Émet `DEATH_EVENT { serverName, channelId, victimName, victimDiscordId, killerName,
  killerDiscordId, cause, distance }`.
- Réponse `{ ok:true, death:{...résumé...} }`.

## Bus / Bot / Embed
- `shared/bus.js` : `DEATH_EVENT`.
- `bot/lib/embeds.js` : `deathEmbed(payload)` — « ☠️ Victim killed by Killer » + cause/distance,
  victime/tueur en mention si lié.
- `bot/bot.js` : `bus.on(DEATH_EVENT, ...)` → poste dans le canal (ignore si pas de canal).

## Plugin `RustLinkRelay.cs` v0.4.0 (non unit-testable → fixture-contrat)
- Config `Relay player deaths` (bool, **défaut false**).
- `OnPlayerDeath(BasePlayer victim, HitInfo info)` : ignore NPC ; extrait killer via
  `info.InitiatorPlayer`, cause via `info.damageTypes.GetMajorityDamageType()`, distance
  via `info.ProjectileDistance` (tout en try/catch — drift Rust). POST `/webhook/death`.

## Tests (node:test)

- `tests/backend/death.test.js` :
