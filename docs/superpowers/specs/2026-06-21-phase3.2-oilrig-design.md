# Phase 3.2 — Auto-détection du hack de crate Oil Rig — Design

Date : 2026-06-21 · Branche : `feat/phase3.2-oilrig`

## Objectif
Quand un joueur lance le **hack de la crate verrouillée** sur le Small/Large Oil Rig,
le plugin poste automatiquement `oil_rig_small` / `oil_rig_large` (status `called`) avec
un compte à rebours jusqu'à l'ouverture (~15 min) — sans qu'un joueur ait à taper `!small`.

## Pourquoi c'est 100% plugin (aucun changement API)
L'API gère déjà `oil_rig_small`/`oil_rig_large` (catalogue + webhook + timer + notif).
La 3.2 n'ajoute qu'un **hook Oxide** dans le plugin qui appelle le `PostEvent` existant.
→ **Aucune surface unit-testable** côté JS ; la suite reste à 23 tests.

## Mécanique (plugin v0.3.0)
- Hook `void OnCrateHack(HackableLockedCrate crate)` (déclenché au **début** du hack).
- Filtre **Oil Rig uniquement** : `WhichOilRig(pos)` parcourt `TerrainMeta.Path.Monuments`,
  garde celui dont `IsInBounds(pos)` est vrai, et mappe le nom :
  `oilrig_1` → `oil_rig_small`, `oilrig_2` → `oil_rig_large`. Sinon → ignoré
  (crate larguée par l'hélico / le cargo hors monument = pas un Oil Rig).
- Durée : `requiredHackSeconds - hackSeconds` (≈ 900 s au lancement) ; fallback
  `ReportMinutes["oil_rig_small/large"]` (15 min) puis 900 s si lecture impossible.
- Envoi : `PostEvent(ev, "called", now, now+remaining, source:null, reporter:null)`
  → événement **auto** (l'API tag `source='webhook'`). Pas un report joueur.
- Config : `Auto-detect Oil Rig crate hacks` (bool, **défaut true**). Logs `Debug` pour
  tracer ce qui est détecté/ignoré sur le serveur.

## Risques / drift (Rust update mensuel)
- Le nom du hook (`OnCrateHack`), le type `HackableLockedCrate` et ses champs
  (`requiredHackSeconds`, `hackSeconds`) peuvent changer → tout est wrappé en try/catch
  avec fallback, et le code renvoie tôt si `TerrainMeta`/monuments indisponibles.
- Les noms de monuments (`oilrig_1`/`oilrig_2`) peuvent varier selon la version → matching
  permissif (`Contains`), à confirmer en live via `Debug`.

## ⚠️ Vérification
**Non vérifiable ici** : pas de serveur Rust. À valider en live —
1. charger le plugin (Oxide/Carbon), activer `Debug`,
2. hacker une crate sur Small puis Large Oil Rig,
3. confirmer le log « Oil Rig crate hack detected: oil_rig_small/large » + la notif Discord
   avec le bon compte à rebours,
4. vérifier qu'une crate hackée hors Oil Rig (heli/cargo) est bien **ignorée**.
