# Phase 8.1 — Commandes in-game équipe & infos — Design

Date : 2026-06-23 · Branche : `feat/phase8.1-ingame-team` · Suit la Phase 7 (socle Rust+).

## Problème

La Phase 7 a posé le socle Rust+ avec juste `!pop`/`!time`. Il reste le gros des commandes
in-game attendues (cf. liste du bot concurrent) : infos équipe (`!online !offline !alive
!prox`), relais (`!bot`), leadership (`!leader`), et requêtes d'events depuis le chat
(`!cargo !small !large !heli`). **Toutes sont *stateless*** (un appel Rust+ → une réponse en
team chat) → pure extension de la table de dispatch du routeur, sans nouveau mécanisme.

## Décision — périmètre (volontairement borné)

Phase 8.1 = **commandes in-game stateless uniquement**. On étend `rustplus/router.js` et on
ajoute 2 appels promisifiés à `connection.js`. Reporté car ce sont des **mécanismes distincts** :

- **8.2 — Poller d'état d'équipe** : `getTeamInfo` sondé en boucle → diff online/position →
  annonces Discord (« connexions », « détecteur d'absence ») **+ `!afk`** (online mais
  immobile = nécessite l'historique de position dans le temps, donc pas stateless).
- **8.3 — Silence & alarmes** : `!silence/!resume` (mute du bot) + `!alarm/!remain/!stop`
  (timers planifiés) — état + persistance + scheduler.

## Contrats Rust+ (lecture `rustplus.proto`)

- `getTeamInfo()` → `response.teamInfo = { leaderSteamId, members[] }`, chaque
  `Member = { steamId, name, x, y, isOnline, spawnTime, isAlive, deathTime }`
  (`x,y` = position monde en mètres ; `spawnTime` = unix du spawn courant).
- `promoteToLeader(steamId)` → requête `{ promoteToLeader: { steamId } }`.
- Distance entre 2 joueurs = `Math.hypot(dx, dy)` sur leurs `x,y`.

## Commandes (toutes répondent via `sendTeamMessage`)

| Commande | Source | Réponse (exemple) |
|---|---|---|
| `!online` | members `isOnline` | `🟢 En ligne (2) : Bob, Alice` |
| `!offline` | members `!isOnline` | `⚫ Hors ligne (1) : Carl` |
| `!alive` | members `isAlive`, **min `spawnTime`** | `⏳ Plus longue vie : Bob (3h 12m)` |
| `!prox` / `!proximity` | positions vs **l'appelant** | `📍 Alice 142m · Carl 1.2km` |
| `!bot <msg>` | — | renvoie `<msg>` tel quel en team chat (relais) |
| `!leader [name]` | `promoteToLeader` | promeut l'appelant, ou le membre `name` |
| `!cargo` `!small` `!large` `!heli` | timer actif (DB) | `🚢 Cargo Ship — dans 12m` / `aucun timer actif` |

Règles :
- **`!prox`** : distances depuis **l'appelant** (son `steamId` = `msg.steamId`) vers chaque
  autre membre online. Appelant introuvable dans l'équipe → message d'erreur court.
- **`!alive`** : parmi les `isAlive`, le plus petit `spawnTime` (= vie la plus longue) ;
  durée = `now - spawnTime`, formatée (`formatCountdown`-style).
- **`!leader`** sans nom → promeut **l'appelant** (`msg.steamId`). Avec nom → cherche le membre
  dont `name` correspond (insensible casse), promeut son `steamId` ; introuvable → erreur.
- **`!bot`** → renvoie le texte après `!bot ` (vide → petit usage). Garder simple (pas d'anti-abus
  ici ; le cooldown/permissions viendront avec le poller si besoin).
- **Events** : la clé (`cargo/small/large/heli`) passe par `resolveEvent()` (alias déjà gérés) →
  `Timers.getByType(serverId, key)` → `eventEmoji`+`eventLabel`+`formatCountdown(expires_at)`.
  Pas de timer / expiré → `aucun timer actif`. `serverId` vient de `client.serverId`.

## Changements de code (TDD)

- `rustplus/connection.js` : `getTeamInfoAsync()` (`sendRequestAsync({getTeamInfo:{}})→res.teamInfo`),
  `promoteToLeaderAsync(steamId)` (`{promoteToLeader:{steamId}}`).
- `rustplus/teamFormat.js` (**pur**, testable sans socket) : `formatOnline`, `formatOffline`,
  `formatAlive`, `formatProx(teamInfo, fromSteamId)`. Réutilise un helper durée local.
- `rustplus/router.js` :
  - `parseCommand` → renvoie aussi les **args** (texte après le 1er token). `handleTeamMessage`
    passe `(client, msg, args)` aux handlers (rétro-compatible : `!pop`/`!time` ignorent `args`).
  - Nouvelles entrées dans `COMMANDS` (+ alias `!proximity`→`!prox`). Handlers d'events générés
    en boucle sur `{cargo,small,large,heli}`.
- `backend/models/timer.js` : `getByType(serverId, eventType)` (1 ligne `.get()`).
- Réutilise `shared/events.js` (`resolveEvent`/`eventLabel`/`eventEmoji`) + `shared/time.js`
  (`formatCountdown`, `nowUnix`). Aucun changement DB/schéma, aucun nouveau secret.

## Tests

- `tests/rustplus/teamFormat.test.js` : online/offline (tri, compte, « personne »),
  `alive` (min spawnTime, ignore les morts, « personne en vie »), `prox` (distances m/km,
  appelant exclu, appelant absent → null/err).
- `tests/rustplus/router.test.js` (étendu, fake client) : chaque commande → bon texte ;
  `!bot foo bar` → renvoie `foo bar` ; `!leader` → `promoteToLeaderAsync(appelant)` ;
  `!leader Bob` → promeut le `steamId` de Bob ; nom inconnu → erreur, pas de promote ;
  `!cargo` avec timer seedé (DB :memory:) → compte à rebours ; sans timer → `aucun timer actif`.
- `parseCommand` : extraction `{cmd,args}`. Anti-boucle (self steamId) déjà couvert, inchangé.
