# Phase 8.5 — Commande `/map` (image + events avec cases) — Plan

Date : 2026-06-24 · Branche : `claude/hopeful-hypatia-uyyjwp` · Suite des Phases 8.2–8.4.

Objectif : exposer la carte en commande Discord. `/map [server]` poste **l'image de la carte**
(`getMap`, déjà mise en cache à la connexion pour les Oil Rigs) + la liste des **events live**
(`getMapMarkers`) avec leur **référence de grille** (ex. `🚢 Cargo Ship — G7`).

## Idée centrale
- La maths de grille (fiddly) est isolée dans un module **pur** `rustplus/grid.js` et testée.
- `mapSize` (taille monde) vient de `getInfo().mapSize` ; les coords markers sont en unités
  monde [0, mapSize], origine sud-ouest. Colonnes A,B,… O→E ; numéros N→S à partir de 0.
- L'image (`AppMap.jpgImage`) est jointe en pièce jointe Discord ; best-effort (absente si
  `getMap` indispo).

## Checklist
- [x] `rustplus/grid.js` (pur) : `worldToGrid(x, y, mapSize)` → `"G12"` ou `null` (hors grille /
      mapSize inconnu) ; `columnLetters` 0-indexé (A…Z, AA…) ; `describeMapMarkers(markers,
      mapSize)` → lignes libellées+case, triées par type, types inconnus ignorés.
- [x] `tests/rustplus/grid.test.js` : coins/inversion N-S, passage Z→AA, hors carte, NaN/0,
      describe (tri par type, hors grille, enum string, vide).
- [x] `bot/lib/embeds.js` : `mapEmbed(server, { mapSize, markers, image })` → `{ embed, files }`
      (image en `AttachmentBuilder` → `attachment://map.jpg`).
- [x] `bot/commands/map.js` : `/map [server]` → `resolveServerOrReply` + `requireConnection`
      (comme `/pop`), `deferReply`, `Promise.all(getInfo, getMapMarkers)` + image cachée.
- [x] `rustplus/connection.js` : cache `this.map` dans `_loadMap` (ex-`_loadOilRigs`) pour que
      `/map` réutilise l'image sans re-fetch.
- [x] README (table des commandes + roadmap 8.5). Pas de nouvel env.
- [x] `npm test` vert (152) + `npm run lint` clean.

## Notes
- **Image en cache** : `getMap` est lourd (jpg) et statique sur un wipe → fetché une fois à la
  connexion (déjà fait pour les rigs), réutilisé par `/map`. Fallback fetch live puis sans image.
- **À valider en live** : le libellé exact de la grille (formule communautaire, cellule
  ≈146,29 u). Bascule de l'origine des numéros = 1 ligne si un serveur diffère.
- **Sans overlay** : v1 ne dessine pas les markers sur l'image (évite une dépendance `canvas`
  native lourde) — l'image brute + la liste de cases suffisent et restent légères.
