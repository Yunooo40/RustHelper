# Phase 8.2 — Détection d'events sans plugin (map markers Rust+) — Plan

Date : 2026-06-24 · Branche : `claude/hopeful-hypatia-uyyjwp`

Objectif : annoncer **Cargo / Heli / Chinook** en live sur **n'importe quel serveur paired,
sans le plugin Oxide**, en pollant `getMapMarkers` et en diffant les snapshots. Les events
passent par le **même pipeline que le webhook** (historique + timer + embed Discord), donc zéro
nouveau chemin de notification. TDD sur le **pur** (diff des markers), socket validée en live.

## Notes
- **Seed anti-spam** : sans seeding, un reboot annoncerait « Cargo spawned » pour un cargo déjà
  présent. Le 1er poll réussi enregistre juste l'état ; idem après une reconnexion brève.
- **Double notif** : si le plugin Oxide rapporte déjà Cargo/Heli/Chinook, mettre
  `RUSTPLUS_MARKERS_ENABLED=false`. Documenté.
- **Pas de timer** : les markers donnent une position live, pas un respawn → `nextRespawn=null`
  (notif seule, comme un webhook sans `next_respawn`). `/status` / `!cargo` restent alimentés par
  le plugin si présent.
